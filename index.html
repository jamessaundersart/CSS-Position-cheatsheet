CSS Property Cheatsheet: Position

Here is a quick guide to the Position Property and its most common values.

The position property is a CSS property that allows us to change where an object (or element) appears on the webpage. Listed below are the most common values used to determine where an element can be placed.

Static

The element behaves normally, this is the default value. Note: the top, left, bottom, right and z-index properties have no effect.

In this example the box stays in its default position.

Relative

The element behaves normally however it is then offset by the value of any subsequent top, left, bottom, and right properties. This might mean that it will overlap other elements, the stacking order can be set by the value of the z-index.

In this example the box is offset by a value of top: 40px and left: 40px. from its default position.

Absolute

The element is taken out of the document flow and then takes a new position relative to the closest parent container that also has a defined position: other than static. Its final position is then determined by the values of top, left, bottom, and right.

In this example the box is taken out of the document flow (meaning the other boxes move together to fill the void) and then takes its position relative to the closest positioned ancestor, in this case the surrounding box which has the property position: relative. Here the values are top: 40px and left: 400px adjust the browser window and watch as the box stays 400px from the left as the surrounding box's width changes.

Fixed

The element is taken out of the document flow and placed in a fixed position on the viewport. This means it stays put as the webpage is scrolled.

In this example, again the box is taken out of the document flow causing the remaining boxes to close up the gap. The box is then fixed to the viewport in this case with the values bottom: 0px and right: 0px causing the box to be fixed in the bottom right hand corner of the viewport.

Fixed Box

Sticky

The element behaves normally until the box reaches the position described by the top, bottom, left or right properties. When this happens the box 'sticks' to that point in much the same way that a 'fixed' element would.

In this example the value described is top: 50px. This means that when the box is 50px from the top of its container, in this case the viewport, it sticks to that position.

Keep scrolling to see 'sticky' in effect.
